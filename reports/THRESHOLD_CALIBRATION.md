# Per-Class Threshold Calibration — Nested LOPO

Date: 2026-04-18  ·  Protocol: nested leave-one-person-out (35 outer × 34 inner).

## TL;DR

**VERDICT:** calibration delivers +0.0046 weighted F1 but bootstrap P(Δ>0)=0.637 (<0.95) and 95% CI=[-0.0242,+0.0315] includes 0. Treat as **noise**, not ship-worthy.

## Protocol

**Nested LOPO (honest, no leakage):**

```
for each outer person P in 1..35:
    outer_train_persons = {1..35} \ {P}           # 34 persons
    outer_eval = scans of P                         # 2–12 scans

    # INNER: build OOF on outer_train WITHOUT P
    for each inner person Q in outer_train_persons:
        inner_train = outer_train_persons \ {Q}    # 33 persons
        fit 3-member v4 on inner_train
        predict softmax on Q's scans → inner_OOF

    τ* = argmax_τ weighted_F1(argmax(inner_OOF − τ), inner_y)
         (coordinate-ascent grid search, τ ∈ {0.05, …, 0.95})

    fit 3-member v4 on ALL of outer_train_persons (with Q=P excluded)
    predict softmax on P's scans
    apply τ*: pred_P = argmax(softmax_P − τ*)

concat pred_P over all 35 outer folds → final honest metrics
```

**Leakage check (red-team self-audit):**
1. Inner OOF for outer fold P is generated by LR heads trained only on `outer_train_persons \ {Q}`, which never contains P → thresholds τ* are learned without any information from P.
2. Outer softmax for P is from a model trained on `outer_train_persons` (= everyone ≠ P) — standard LOPO, already leakage-free.
3. Concatenation of outer predictions into the final metric never reuses the inner OOF probabilities (those are discarded after τ* is picked).
4. The cache file `v4_oof_predictions.npz` was **NOT** used to tune thresholds — we rebuilt all softmaxes from scratch with the nested protocol. Flat-OOF tuning (using v4_oof_predictions.npz for both tuning and eval) is the leaky recipe rejected in earlier waves (see `reports/RED_TEAM_*`).
